# 📊 任务复杂度分析 - 快速入门

## 🎯 核心发现

你说得对 - 我之前的方案过于单一。我现在理解了：

### ✅ 简单任务（点赞、签到、导航）
- 只需 **1-2 个动作**
- **无需判断**，一步到位
- 不应该为每个都单独设计提示词
- 可以用 **1 个统一的简单提示词** 处理所有这类任务

### ✅ 复杂任务（看广告、看视频）
- 需要 **多步骤 + 多判断**
- 需要 **根据不同情况采取不同操作**
- 必须用 **针对性的深度优化提示词**

## 📋 当前问题

### 看广告
```
问题：虽然有警告"不要提前退出"，但 AI 仍然提前点击"跳过"

缺失：
❌ 如何识别倒计时（5秒、4秒...0）
❌ "跳过" vs "关闭" 按钮的区别和出现时机
❌ 应该等到什么时候才能点击
```

### 看视频
```
问题：AI 无法识别哪个视频有奖励

缺失：
❌ 奖励标签的各种样式（金币图标、"奖励"文字、✅标记等）
❌ 有/无奖励时的不同操作策略
❌ 如何判断一个视频已经看完
```

## 🔍 解决方案

### 第 1 步：收集真实数据
```bash
python scripts/monitor_task_execution.py watch_ad
python scripts/monitor_task_execution.py watch_video
```

### 第 2 步：分析 AI 的决策过程
查看执行输出，回答：
- AI 是否识别了倒计时？
- AI 是否做出了有/无奖励的判断？
- 在哪一步失败了？

### 第 3 步：根据数据改进提示词
```python
# 简单任务：统一为 1 个（200 字）
# 看广告：优化为 600-800 字（加入倒计时识别、按钮逻辑）

```
